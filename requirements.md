# Requirements Document - Exam-Sarthi

## Introduction

Exam-Sarthi is an intelligent study planner web application designed to help students optimize their exam preparation through AI-powered analysis and personalized scheduling. The system analyzes uploaded exam syllabus PDFs and Previous Year Question papers using AWS Bedrock Agents to generate customized 3-day crash course timetables, provide intelligent Q&A capabilities, and create targeted flashcards for high-priority topics.

## Project Overview

**Domain**: Educational Technology / AI-Powered Study Planning
**Technology Stack**: React.js frontend, FastAPI backend, AWS Bedrock (Claude 3.5 Sonnet), Amazon DynamoDB, Amazon S3
**Target Users**: Students preparing for exams who need structured, AI-optimized study schedules

## Glossary

- **System**: The Exam-Sarthi web application
- **Smart_Scheduling_Agent**: AWS Bedrock Agent that analyzes syllabus and generates personalized study schedules
- **RAG_System**: Retrieval-Augmented Generation system using Amazon Bedrock Knowledge Bases
- **Flashcard_Generator**: Component that automatically creates revision flashcards
- **PDF_Processor**: Component that handles PDF upload and processing
- **User_Interface**: React.js frontend application
- **API_Gateway**: FastAPI backend service
- **Knowledge_Base**: Amazon Bedrock Knowledge Base storing processed PDF content
- **Schedule_Database**: DynamoDB database storing user schedules and preferences
- **Student**: End user of the application
- **Syllabus_PDF**: PDF document containing exam syllabus information
- **PYQ_PDF**: Previous Year Question paper PDF document
- **Crash_Course**: 3-day intensive study schedule generated by the system
- **Weightage_Analysis**: AI analysis of topic importance based on syllabus and PYQ patterns

## Requirements

### Requirement 1: PDF Upload and Processing

**Priority**: High
**User Story**: As a student, I want to upload my exam syllabus and Previous Year Question papers as PDFs, so that the system can analyze them to create my personalized study plan.

#### Acceptance Criteria

1. WHEN a student uploads a PDF file, THE System SHALL validate the file format and size (max 10MB per file)
2. WHEN a valid PDF is uploaded, THE PDF_Processor SHALL extract text content and store it in the Knowledge_Base
3. WHEN PDF processing is complete, THE System SHALL notify the student of successful upload
4. WHEN an invalid file is uploaded, THE System SHALL display an error message and prevent processing
5. THE System SHALL support simultaneous upload of multiple PDF files (up to 5 files per session)

### Requirement 2: Smart Scheduling Agent

**Priority**: High
**User Story**: As a student, I want an AI agent to analyze my syllabus and create a personalized 3-day crash course schedule, so that I can focus on high-priority topics efficiently.

#### Acceptance Criteria

1. WHEN syllabus analysis is requested, THE Smart_Scheduling_Agent SHALL analyze uploaded PDFs using Claude 3.5 Sonnet
2. WHEN analyzing content, THE Smart_Scheduling_Agent SHALL identify topic weightages based on syllabus structure and PYQ patterns
3. WHEN weightage analysis is complete, THE Smart_Scheduling_Agent SHALL generate a 3-day study schedule with time allocations
4. WHEN generating schedules, THE System SHALL consider topic difficulty, weightage, and optimal learning sequences
5. THE Smart_Scheduling_Agent SHALL provide reasoning explanations for schedule recommendations

### Requirement 3: RAG-based Question Answering

**Priority**: High
**User Story**: As a student, I want to ask questions about my study materials and get accurate answers from my uploaded PDFs, so that I can clarify doubts quickly during preparation.

#### Acceptance Criteria

1. WHEN a student submits a question, THE RAG_System SHALL search the Knowledge_Base for relevant content
2. WHEN relevant content is found, THE RAG_System SHALL generate contextual answers using Claude 3.5 Sonnet
3. WHEN providing answers, THE System SHALL cite specific PDF sources and page references
4. WHEN no relevant content is found, THE System SHALL inform the student and suggest alternative queries
5. THE RAG_System SHALL maintain conversation context for follow-up questions

### Requirement 4: Flashcard Generation

**Priority**: Medium
**User Story**: As a student, I want the system to automatically create flashcards for important topics, so that I can efficiently review key concepts during my preparation.

#### Acceptance Criteria

1. WHEN high-priority topics are identified, THE Flashcard_Generator SHALL create question-answer flashcard pairs
2. WHEN generating flashcards, THE System SHALL prioritize topics based on weightage analysis results
3. WHEN flashcards are created, THE System SHALL organize them by subject and difficulty level
4. THE Flashcard_Generator SHALL create a minimum of 20 flashcards per high-priority topic
5. WHEN displaying flashcards, THE User_Interface SHALL provide interactive review functionality with progress tracking

### Requirement 5: User Interface and Experience

**Priority**: High
**User Story**: As a student, I want a clean and intuitive interface that makes it easy to upload files, view my schedule, and access study materials, so that I can focus on learning rather than navigating the application.

#### Acceptance Criteria

1. WHEN a student accesses the application, THE User_Interface SHALL display a clean, responsive dashboard
2. WHEN uploading files, THE User_Interface SHALL provide drag-and-drop functionality with progress indicators
3. WHEN viewing schedules, THE User_Interface SHALL display timetables in an easy-to-read calendar format
4. WHEN using flashcards, THE User_Interface SHALL provide smooth card-flipping animations and progress tracking
5. THE User_Interface SHALL be fully responsive and work seamlessly on desktop and mobile devices

### Requirement 6: Data Persistence and Management

**Priority**: Medium
**User Story**: As a student, I want my study schedules and progress to be saved automatically, so that I can access my personalized plan across different sessions and devices.

#### Acceptance Criteria

1. WHEN a schedule is generated, THE System SHALL store it in the Schedule_Database with user identification
2. WHEN a student returns to the application, THE System SHALL retrieve and display their saved schedules
3. WHEN progress is made on flashcards, THE System SHALL update completion status in real-time
4. THE Schedule_Database SHALL maintain data consistency and handle concurrent user access
5. WHEN storing user data, THE System SHALL implement proper data encryption and privacy protection

### Requirement 7: AWS Bedrock Integration

**Priority**: High
**User Story**: As a system administrator, I want the application to leverage AWS Bedrock services effectively, so that we can provide reliable AI-powered features with optimal performance and cost efficiency.

#### Acceptance Criteria

1. WHEN processing requests, THE System SHALL use Claude 3.5 Sonnet model through AWS Bedrock
2. WHEN managing knowledge bases, THE System SHALL utilize Amazon Bedrock Knowledge Bases for efficient retrieval
3. WHEN implementing agents, THE System SHALL use Strands SDK for agentic workflow orchestration
4. THE System SHALL implement proper error handling for AWS service failures
5. THE System SHALL optimize API calls to minimize costs while maintaining performance

### Requirement 8: Performance and Scalability

**Priority**: Medium
**User Story**: As a student, I want the application to respond quickly to my requests and handle multiple users simultaneously, so that I can study efficiently without technical delays.

#### Acceptance Criteria

1. WHEN processing PDF uploads, THE System SHALL complete analysis within 2 minutes for files up to 10MB
2. WHEN generating schedules, THE Smart_Scheduling_Agent SHALL provide results within 30 seconds
3. WHEN answering questions, THE RAG_System SHALL respond within 10 seconds for standard queries
4. THE System SHALL support at least 100 concurrent users without performance degradation
5. WHEN system load increases, THE API_Gateway SHALL implement rate limiting to maintain service quality

### Requirement 9: Error Handling and Reliability

**Priority**: Medium
**User Story**: As a student, I want the system to handle errors gracefully and provide clear feedback when issues occur, so that I understand what went wrong and how to proceed.

#### Acceptance Criteria

1. WHEN AWS services are unavailable, THE System SHALL display appropriate error messages and retry mechanisms
2. WHEN PDF processing fails, THE System SHALL provide specific error details and suggested solutions
3. WHEN network issues occur, THE User_Interface SHALL show connection status and offline capabilities where possible
4. THE System SHALL log all errors for debugging while protecting user privacy
5. WHEN critical errors occur, THE System SHALL gracefully degrade functionality rather than complete failure

### Requirement 10: Security and Privacy

**User Story:** As a student, I want my uploaded documents and personal study data to be secure and private, so that I can trust the system with my sensitive academic materials.

#### Acceptance Criteria

1. WHEN files are uploaded, THE System SHALL encrypt data in transit and at rest
2. WHEN storing user data, THE Schedule_Database SHALL implement access controls and audit logging
3. WHEN processing PDFs, THE System SHALL ensure uploaded content is not shared between users
4. THE System SHALL implement session management with secure authentication tokens
5. WHEN users request data deletion, THE System SHALL completely remove all associated data within 24 hours
